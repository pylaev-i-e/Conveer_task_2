## Описание настроенных джобов (что за джобы, какие GitHub Actions были использованы, какие ивенты триггерят джобы, зависимости между джобами)

---

### 1. Job: `install`

- **Назначение:** подготовка окружения — установка Java и предзагрузка Maven-зависимостей для ускорения последующих шагов.
- **Используемые GitHub Actions:**
  - `actions/checkout@v4` — клонирование репозитория
  - `actions/setup-java@v4` — установка JDK 17 (Temurin)
- **Триггер:** запускается при любом `push` или `pull_request`.
- **Зависимости:** нет входящих зависимостей.

---

### 2. Job: `build`

- **Назначение:** компиляция исходного кода без запуска тестов.
- **Используемые GitHub Actions:**
  - `actions/checkout@v4`
  - `actions/setup-java@v4`
- **Триггер:** запускается при любом `push` или `pull_request`.
- **Зависимости:** зависит от успешного завершения job `install`.

---

### 3. Job: `test`

- **Назначение:** запуск юнит-тестов на четырёх операционных системах для кроссплатформенной проверки.
- **Используемые GitHub Actions:**
  - `actions/checkout@v4`
  - `actions/setup-java@v4` с кэшированием Maven-зависимостей (`cache: 'maven'`)
  - `EnricoMi/publish-unit-test-result-action@v2` — публикация отчёта (только на Ubuntu)
- **Триггер:** запускается при любом `push` или `pull_request`.
- **Зависимости:** зависит от job `install` → `needs: install`

---

### 4. Job: `deploy`

- **Назначение:** развёртывание приложения на Heroku.
- **Используемые GitHub Actions:**
  - `actions/checkout@v4` с `fetch-depth: 0` — полный клон (не shallow), чтобы Heroku принял `git push`
- **Триггер:** только при событии `push` в ветку `master`.
- **Зависимости:** зависит от успешного завершения jobs `build` и `test`.
